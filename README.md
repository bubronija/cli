## Задание 
Создать минимальное CLI-приложение и сделать его настраиваемым.

1. Источником настраиваемых пользователем параметров являются опции 
командной строки. 
2. К настраиваемым параметрам относятся: – Имя анализируемого пакета. – URL-адрес репозитория или путь к файлу тестового репозитория. – Режим работы с тестовым репозиторием. – Версия пакета. – Максимальная глубина анализа зависимостей. – Подстрока для фильтрации пакетов. 
3. (только для этого этапа) При запуске приложения вывести все параметры, 
настраиваемые пользователем, в формате ключ-значение. 
4. Реализовать и продемонстрировать обработку ошибок для всех параметров. 
5. Результат выполнения этапа сохранить в репозиторий стандартно 
оформленным коммитом.

## Описание 
Данный проект представляет собой минимальный прототип CLI-инструмента для анализа пакетов. Приложение через аргументы командной строки позволяет пользователь указать имя пакета, репозиторий, режим работы, версию, максимальную глубину анализа и фильтр пакетов.
На текущем этапе в прототипе реализован функционал позволяющий:
1. Парсинг и проверку пользовательских параметров на корректность.
2. Вывод всех переданных параметров в формате ключ значение.
   
Прототип демонстрирует функции, необходимые для будущего инстурмента анализа зависимостей пакетов.
